CURRENT PROBLEMS
----------------
1. REFERENCE.HTML IS OVERLOADED
   A single flat page lists 40+ items across 13 unrelated topic sections.
   Electronics hardware, machine shop manuals, wood shop guides, network config,
   Tesla subscriptions, and Youtube playlists all live in the same list.
   This is unscannable and will only get worse as content grows.

2. INDEX.HTML AND TOOLS.HTML OVERLAP
   Both pages contain links to external tools and services. The home page
   has LLMs, web mail, search engines, and misc tools. The tools page has
   Grafana, Prometheus, Autodesk, and Bambu Labs. There is no principled
   distinction between the two.

3. LOCALLY-HOSTED APPS ARE BURIED
   The fitness tracker and recipe collection are only discoverable via tools.html.
   They are not in the nav bar and have no visual prominence.

4. REFERENCE/ HTML FILES ARE ORPHANED
   macStudio.html, mitchellNet.html, and testbenchEthernet.html are standalone
   reference documents in the reference/ subfolder, only reachable from links
   inside reference.html. They are not surfaced in any consistent way.

5. PROJECTS.HTML IS SPARSE
   The page has four project categories; one ("Machine Shop") says "Nothing yet."
   The methodology content is just a PDF link with no surrounding context.

6. NETWORK/SERVER INFO IS SPLIT ACROSS TWO PAGES
   about.html lists server hardware, IP addresses, OS, and network details.
   reference.html has a "Network & Server Administration" section with iMac
   server config, MQTT broker, and Ubuntu GUI docs. These belong together.

7. DUPLICATE LINK IN REFERENCE.HTML
   "AI Infrastructure plan" appears in both the "AI & Machine Learning" section
   and the "ESP32 Microcontroller" section. The ESP32 entry also has a wrong
   path (/AI_Dev_Environment_v3.pdf instead of /reference/AI_Dev_Environment_v3.pdf).

8. DEPRECATED CONTENT LEFT IN PLACE
   index.html lists two deprecated sites and two deprecated GitHub pages with
   inline red "Deprecated" labels. These add noise and could be archived.

9. TYPOS ACROSS MULTIPLE FILES
   - "recipies" should be "recipes" (folder name, file name, and page title)
   - "pracrtices" in projects.html
   - "hany" in tools.html
   - "produciton" in about.html

ORGANIZING PRINCIPLE
--------------------
Group content by the domain it serves, not by file type. The site covers four
distinct domains that a user visits for different reasons:

  (A) ENGINEERING — electronics, embedded systems, software, AI/ML, PCB design
  (B) WORKSHOP     — machine shop, wood shop, 3D printing/scanning, electrical
  (C) INFRASTRUCTURE — MitchellNET network, servers, monitoring, home automation
  (D) MAKING / PROJECTS — active and past maker projects

The current nav (Home | Tools | Reference | Projects | About) does not map to
these domains. "Reference" is a container, not a topic. "Tools" mixes local apps
with external links.

PART 3: PRESENTATION RECOMMENDATIONS
======================================

1. REPLACE FLAT LISTS WITH CARD GRIDS ON HUB PAGES
   The engineering, workshop, and infrastructure index pages each have 6-10
   topic sections. Present each section as a card with:
     - Icon or emoji representing the topic
     - Section title (e.g., "PCB Design")
     - 1-line description
     - Expandable list of links (or inline if fewer than 4)
   This makes the page scannable at a glance without scrolling through a wall
   of <ul> items.

2. ADD COLLAPSIBLE SECTIONS (ACCORDIONS) FOR DENSE CONTENT
   Reference sections with 5+ links (PCB Design has 7, Machine Shop has 7)
   should default to collapsed on page load. A user looking for one specific
   thing can expand only what they need. Simple CSS + minimal JS, no framework.

3. ADD A SEARCH/FILTER BAR ON EACH HUB PAGE
   A text input that filters visible links in real time (JS string match on
   link text + description). This is especially valuable on the engineering hub
   where there are 30+ items. Implementation is ~20 lines of vanilla JS.

4. IMPROVE HOME PAGE LAYOUT
   Replace the plain HTML table with a two-column card layout grouping:
     - Communication (web mail)
     - AI Assistants (LLMs)
     - Search
     - My Presence (GitHub, YouTube, external site)
   Each group gets a header and icon. More scannable than a two-column table.

5. GIVE LOCALLY-HOSTED APPS MORE VISIBILITY
   The fitness tracker and recipes collection are useful daily-driver tools.
   Options:
     (a) Add an "Apps" link to the main nav bar
     (b) Feature them as prominent cards on the Home page
     (c) Surface them in Projects page under a "Hosted Apps" section
   Recommend (b): add two app cards to the Home page above the quick-links table.

6. FIX CONTENT BUGS BEFORE OR DURING MIGRATION
   - Fix duplicate AI Infrastructure link in reference.html ESP32 section
   - Fix broken path: /AI_Dev_Environment_v3.pdf → /reference/AI_Dev_Environment_v3.pdf
   - Rename "recipies" to "recipes" in folder, file, and page title
   - Fix typos: pracrtices, hany, produciton
   - Fix h1-inside-p nesting in about.html
   - Update page <title> tags (currently "reference", "tools" — use sentence case)

7. CONSOLIDATE DUPLICATE EXTERNAL LINKS
   Bambu Labs appears in both index.html and tools.html. Shaper Tools appears
   in index.html and is referenced in reference.html (YouTube). Pick one home
   for each and link from the other, or just remove the duplicate.
