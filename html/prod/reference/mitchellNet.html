MitchellNET — Network Design

Master design specification for the MitchellNET home network rebuild. Covers architecture, component roles, VLAN segmentation, the electronics room aggregation switch, test bench ethernet integration, bill of materials with current Canadian pricing, deployment checklist, and rationale for all major decisions. Location: London, Ontario, Canada.

1. Architectural Overview

The design implements a high-performance, segmented network spanning two rooms with three distinct planes:

  • 10 Gbps Data Plane — connects the Mac Studio, NAS, and core infrastructure via SFP+ DAC cables (rack) or OM3 multimode fibre (inter-room, 20–30 m runs) through the MikroTik CRS317 core switch.
  • 1 Gbps OOB Management Plane — a physically separate TP-Link TL-SG108E provides a recovery lifeline to every infrastructure device's management interface in the server room. Even if the core 10G network suffers a misconfiguration or broadcast storm, this rail remains reachable.
  • 1 Gbps Electronics Room Plane — a TP-Link TL-SG116E (16-port Easy Smart) in the electronics room aggregates all 8 devices in that room (4 bench instruments + 4 other network devices) and sends a single OM3 fibre uplink back to the CRS317 core switch.

Core services (DHCP, DNS, IDS/IPS, VLAN routing) live on the Protectli VP2430 firewall running OPNsense. Application services (MQTT, reverse proxy, monitoring) run as Docker containers on the Ubuntu iMac server.

3. VLAN Segmentation & WiFi SSID Mapping

Every device lands on exactly one VLAN. OPNsense is the gateway for all VLANs and enforces strict inter-VLAN firewall rules. The UniFi APs broadcast all SSIDs simultaneously — the SSID a device connects to determines its VLAN, regardless of which physical AP serves it. The TL-SG116E in the electronics room trunks all VLANs on its uplink port; individual ports are assigned to the appropriate access VLAN.

  • VLAN 10 Management — 10.10.10.0/24. Infrastructure OOB interfaces only. Not reachable from any other VLAN.
  • VLAN 20 Storage — 10.10.20.0/24. Mac Studio and NAS only. MTU 9000 (jumbo frames) for maximum NFS/SMB throughput.
  • VLAN 30 IoT — 10.10.30.0/24. ESP32 devices, smart home, seasonal devices. Firewall allows only outbound to Mosquitto port 1883. Client isolation on IoT SSID.
  • VLAN 40 DMZ — 10.10.40.0/24. Raspberry Pi. Inbound HTTPS only. Cannot initiate connections to LAN.
  • VLAN 50 Lab / Electronics Room — 10.10.50.0/24. All four bench instruments. 4 general devices in electronics room if they need lab-level isolation. Mac Studio permitted inbound SCPI on port 5025 only.
  • VLAN 99 Guest — 10.10.99.0/24. Internet access only. Blocked from all internal resources. Maps to AgingApprentice-Guest SSID.

4. Electronics Room Network NEW

What changed: The TP-Link Archer A6 access point is retired. Its four LAN-connected devices now connect to the TL-SG116E. WiFi in the electronics room is provided by one of the three UniFi U6 Pro APs. All 8 ethernet devices in the room (4 bench instruments + 4 general devices) feed into a single 10 GbE OM3 fibre uplink back to the server room CRS317.

Why TL-SG116E instead of replacing the OOB switch

The TL-SG116E is a 16-port 802.1Q-capable Easy Smart switch — well-suited for in-band aggregation of multiple devices onto a single trunked uplink. It is not appropriate as the server-room OOB switch, which requires a dedicated management-only rail isolated from all production traffic. The TL-SG108E in the server room stays as-is.

None of the electronics room devices (instruments, general room devices) need OOB connectivity — OOB is reserved for infrastructure devices that need emergency management when the core network is down. If a future device in the electronics room gains an IPMI or serial console port, a single Cat6 run from the TL-SG116E back to the TL-SG108E is all that is needed.

Inter-Room Fibre Run — Electronics Room

Like the Mac Studio run, the 20–30 m distance to the server room rules out DAC cables (max ~7 m). An OM3 multimode fibre LC-LC duplex cable carries 10 GbE between the TL-SG116E and the CRS317. The TL-SG116E needs a 10 GbE SFP+ uplink port — the TL-SG116E has standard RJ45 ports only (no built-in SFP+). Two options:

  • Option A (recommended): Use the TL-SG116E's uplink port with a 10GBase-T RJ45 transceiver inserted into the CRS317, and run Cat6A copper (up to 55 m at 10 G). This avoids needing an SFP+ cage on the TL-SG116E side. A Cat6A solid-core pull cable from the electronics room to the server room handles this cleanly.
  • Option B: Add a small media converter (GbE copper → SFP+ fibre) at the electronics room end. Runs 1 GbE aggregate — sufficient for the room's actual traffic (instruments + 4 devices), though not future-proof.
Practical recommendation: The TL-SG116E's total aggregate traffic from 8 GbE devices will rarely exceed 2–3 Gbps simultaneously. Running Cat6A copper (Option A) is simpler and cheaper than pulling OM3 fibre for this room. Reserve the OM3 approach for the Mac Studio run where sustained 10 GbE NAS throughput is critical. Pull the Cat6A and use a 10GBase-T RJ45 SFP+ transceiver (~$40) in a CRS317 port.

5. Test Bench — Ethernet Remote Control

All four bench instruments support remote control over ethernet using SCPI (Standard Commands for Programmable Instruments) over the LXI standard. They now connect through the TL-SG116E on VLAN 50, with the uplink back to the CRS317. The Mac Studio reaches instrument SCPI ports (TCP 5025) via an OPNsense allow rule from VLAN 20.

Example RC time constant experiment: a single script tells the SDG to output a 0.2 Hz square wave, the DP832 to hold exactly 5.000 V, the SDM to log capacitor voltage every 100 ms, and the SDS to capture a screenshot — all without touching any front panel.

What Each Instrument Supports Remotely

  • Siglent SDS1202X-E (Oscilloscope) — capture screenshots, pull waveform data, set timebase/voltage/trigger, automate measurements and log results.
  • Siglent SDG-2042X (Signal Generator) — set frequency, amplitude, waveform type, DC offset; configure sweep sequences; save and recall waveform setups.
  • Siglent SDM3055 (Multimeter) — READ? returns live measurements; log voltage/current/resistance to CSV over time.
  • Rigol DP832 (Power Supply) — set voltage and current limits; enable/disable outputs; log actual output; automate power sequencing.

6. Application Stack (Ubuntu iMac Server)

All services run as Docker containers on the 2019 iMac running Ubuntu Server.

  • Nginx Reverse Proxy — entry point for external web services; traffic isolated to DMZ VLAN 40.
  • Mosquitto MQTT Broker — logic hub for all IoT devices; VLAN 30 allowed on port 1883 only.
  • Grafana + Prometheus + InfluxDB — monitoring dashboards; stores time-series from instruments and IoT devices.
  • LibreNMS — polls MikroTik and all network devices via SNMP over the OOB rail; per-port traffic, SFP temperatures, device health.
  • UniFi Network Application — manages all three U6 Pro APs; VLAN/SSID assignment, client isolation, RF data.

7. Firewall & Security

OPNsense on the Protectli VP2430. Key rules:

  • VLAN 20 (Storage) → Any: allow. Mac Studio and NAS communicate freely.
  • VLAN 30 (IoT) → Mosquitto port 1883 only: restricted. All other IoT traffic blocked.
  • VLAN 40 (DMZ) → LAN: blocked. Inbound HTTPS 443 from internet permitted only.
  • VLAN 50 (Lab) → Mac Studio SCPI port 5025 only: restricted. All instruments isolated from other segments.
  • VLAN 99 (Guest) → LAN: blocked. Internet only.

Suricata IDS/IPS runs in IPS mode on WAN, VLAN 30 (IoT), and VLAN 40 (DMZ) interfaces. Not on VLAN 20 (Storage) — it would throttle NAS throughput on N150-class hardware. WireGuard VPN for remote access with AES-NI hardware offload.

10. Key Design Decisions — Rationale

MikroTik CRS317 vs. Enterprise Alternatives

Enterprise 16-port 10 GbE switches (Cisco, Juniper) cost $3,000–$5,000 CAD and are too loud for a home office. The CRS317 delivers wire-speed 10 GbE for ~$750 CAD, is passively cooled (silent), supports dual-boot SwOS/RouterOS, and exposes deep SNMP telemetry ideal for Grafana dashboards. 20–40 W vs. 150 W+ for equivalent enterprise gear.

Protectli VP2430 vs. Netgate

The VP2430 (launched May 2025, N150 CPU) replaces the previous Netgate 6100 recommendation. It offers 46% higher PassMark, DDR5 RAM, roughly double the VPN throughput, and coreboot open-source firmware — all from a Canadian assembly location at $579 CAD. The Netgate remains valid if OPNsense pre-install on VP2430 is not preferred.

TL-SG116E for Electronics Room vs. Managed Switch

A fully managed switch (UniFi, Cisco SB) is overkill for this room. The TL-SG116E supports 802.1Q VLAN tagging — enough to assign ports to the correct VLAN and trunk back to the CRS317 — at a fraction of the cost. Its 16 ports provide room for additional instruments or devices without touching the server room switch configuration.

OOB Switch Stays as TL-SG108E

The TL-SG116E you found is an excellent electronics room aggregation switch, but is not a replacement for the OOB switch. The OOB rail must be physically separate from production traffic — the TL-SG108E's dedicated role as a management-only switch with known, minimal configuration is the right design. Mixing OOB and production traffic defeats the purpose of having an out-of-band rail.

Cat6A Copper vs. OM3 Fibre for Electronics Room Uplink

The electronics room's 8 devices will never saturate even 1 GbE in aggregate under realistic use. Cat6A copper to the server room (Option A) is simpler, cheaper (~$75 total including transceiver and cable vs. ~$150+ for OM3 + media converters), and supports 10 GbE if needed. OM3 fibre is reserved for the Mac Studio run where sustained 10 GbE NAS editing throughput is the explicit goal.

DAC Cables vs. 10GBase-T (In-Rack)

DAC cables have lower latency, consume less power, and run cooler than 10GBase-T RJ45 transceivers. RJ45 10 GbE modules can thermal-throttle during long renders. For all rack-internal connections DAC is preferred; 10GBase-T adapters are used only where a device lacks an SFP+ port.

OPNsense vs. pfSense

OPNsense: native Suricata integration, more frequent updates, modern GUI, active community. pfSense remains solid if its WireGuard implementation or documentation is preferred. Both are capable.

11. Day 1 Deployment Checklist

Phase 1Bench Prep & Physical Labelling
  • Unbox and inspect all hardware. Verify power cables, mounting ears, and SFP+ modules.
  • Label every cable before installation (e.g. CORE-TO-NAS, OOB-TO-FIREWALL, ELEC-ROOM-UPLINK).
  • Update firmware on all devices before complex configuration begins.
Phase 2OOB Management Rail — Safety Net First
  • Initialize TL-SG108E OOB switch. Set static management IP: 10.10.10.2.
  • Connect MikroTik console/MGMT port → OOB Switch.
  • Connect Synology DS1823xs+ OOB mgmt port → OOB Switch.
  • Connect VP2430 dedicated mgmt port → OOB Switch.
  • Connect Ubuntu iMac USB–Ethernet adapter → OOB Switch.
  • Test: laptop on OOB switch can ping all management interfaces. Do not proceed until all devices have a lifeline.
Phase 3Core VLAN & Data Plane
  • Define VLANs on OPNsense: VLAN 10 (10.10.10.1/24), VLAN 20 (10.10.20.1/24, MTU 9000), VLAN 30 (10.10.30.1/24), VLAN 40 (10.10.40.1/24), VLAN 50 (10.10.50.1/24), VLAN 99 (10.10.99.1/24).
  • Configure SFP+ firewall ↔ MikroTik port as VLAN trunk (tagged for all VLANs).
  • Set MikroTik ports for NAS and Mac Studio to access mode — untagged on VLAN 20.
  • Set MikroTik port for electronics room uplink to trunk mode — all VLANs tagged.
  • Configure MikroTik trunk port to UniFi APs (tagged for VLANs 1, 30, 99).
Phase 4Electronics Room Switch
  • Configure TL-SG116E: ports 1–4 as VLAN 50 access, ports 5–8 as VLAN 1 access, port 9 as trunk for the U6 Pro AP, port 16 as trunk uplink.
  • Run Cat6A from electronics room to server room. Terminate and test with cable tester.
  • Install 10GBase-T transceiver in CRS317 port. Connect Cat6A. Verify link-up at 10G.
  • Power on TL-SG116E. Verify all 8 room devices obtain DHCP leases on correct VLANs.
Phase 5ISP Integration
  • Input Bell b1xxxxxx PPPoE credentials into OPNsense WAN interface.
  • Verify OPNsense shows a public IP (not 192.168.x.x — that means double-NAT).
  • Disable Bell HH3000 WiFi, DHCP, and UPnP completely.
Phase 6Service Migration
  • Assign Ubuntu iMac a static IP on VLAN 10 (management).
  • Verify Nginx serves one test site reachable from the internet via public IP.
  • Verify LibreNMS successfully polls MikroTik via SNMP over OOB rail.
  • Verify Grafana/Prometheus dashboards reachable from Mac Studio.
  • Deploy UniFi Network Application Docker container. Adopt all three U6 Pro APs.
Phase 7Test Bench & NAS Integration
  • Enable DHCP on each instrument: Utility → I/O → LAN → DHCP on.
  • Verify each instrument's built-in web interface loads in Mac browser.
  • Run LXI Tools discovery scan — all four instruments should appear.
  • Add OPNsense firewall rule: VLAN 20 (Mac Studio) → VLAN 50 (instruments) port 5025 allow.
  • Add NAS drives. Configure RAID6 array. Add NVMe SSD cache.
  • Test sustained NAS throughput from Mac Studio: target >800 MB/s sequential read.

12. Risks & Key Considerations

  • Bell HH3000 bridge mode — must be in true PPPoE passthrough; otherwise double-NAT will break VLAN setup and external connectivity.
  • NAS throughput — ensure SSD cache tier or RAID with fast drives. Without caching, spinning disks bottleneck well below 10 G.
  • Suricata throughput — IDS/IPS in full inspection mode reduces throughput on weaker hardware. Tune noisy rule categories (coin mining, DNS, TLS) to maintain line rate.
  • MikroTik learning curve — RouterOS is engineer-first. Start with SwOS; migrate to RouterOS when comfortable with VLAN and routing concepts.
  • TL-SG116E VLAN limits — the TL-SG116E is Easy Smart, not full L2 managed. Confirm that 802.1Q port-based VLAN tagging covers the required configuration before building out; it lacks some advanced features of Omada or UniFi switches.
  • Cat6A pull quality — for the electronics room 10 G copper run, use solid-core Cat6A and terminate carefully. A poor termination will limit you to 1 GbE. Test with a cable certifier if 10 G link-up is inconsistent.
  • IoT MAC tracking — seasonal devices (Christmas lights, candy cane pillars) will get new DHCP leases each season. Track by MAC in Grafana/LibreNMS, not IP.
  • Ontario hydro costs — 24/7 rack draw: CRS317 (~30W) + VP2430 (~15W) + TL-SG108E (~5W) + NAS (~50W idle) + iMac server (~80W) ≈ 180W continuous.
