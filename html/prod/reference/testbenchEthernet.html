Test Bench — Ethernet Remote Control from Mac Studio

All bench instruments support remote control from the Mac Studio over the local network. This page documents the two standards that make this work, the software used to talk to the instruments, what can be done with each one, and how to combine them all into automated test scripts.

How It Works: LXI and SCPI

Two complementary standards work together to make Python talk to the test equipment. They are not competing technologies — LXI is how the computer connects to the instrument, and SCPI is the language they speak once connected.

LXI — LAN eXtensions for Instrumentation

LXI is the standard that governs the physical and network connection. Because each instrument is LXI-compliant, once it is plugged into the network switch it behaves like a standard network device:

  • It automatically gets an IP address via DHCP
  • It hosts a built-in web page you can open in any browser to view status and send basic commands
  • It uses standard TCP/IP so the Mac can discover and connect to it without any proprietary drivers

SCPI — Standard Commands for Programmable Instruments

SCPI (pronounced "skippy") is the universal syntax and command set used to control the instrument. Once Python opens a network connection to an instrument via LXI, it sends plain-text SCPI commands down that connection to tell the instrument exactly what to do.

A typical Python automation flow looks like this:

[ Your Python Script ]
       │
       ▼  (LXI / TCP-IP — connects to instrument at its IP address)
[ Network Connection ]
       │
       ▼  (sends the SCPI text string: ":SOURce1:VOLTage 5.0")
[ Instrument ]  ──▶  Sets Channel 1 to 5 Volts

Software: LXI Tools

The recommended free tool for instrument discovery and interactive control is LXI Tools — open source, runs natively on Mac, and supports all the Siglent instruments on this bench. Install via Homebrew: 

brew install lxi-tools

LXI Tools includes both a command-line tool and a GUI. The GUI lets you discover all instruments on the network, send SCPI commands interactively, capture screenshots from instrument displays, and record data — all from one interface.

To discover all instruments on the bench network:

lxi discover

To grab a screenshot from the oscilloscope directly to the Mac:

lxi screenshot -a 192.168.2.45 screenshot.png

Python Automation with PyVISA

For scripted automation, the pyvisa library is the standard tool. It handles the VISA connection layer so scripts can talk to any instrument with a consistent API regardless of whether it is connected via Ethernet or USB. 

pip install pyvisa pyvisa-py

A minimal example connecting to the oscilloscope:

import pyvisa

rm = pyvisa.ResourceManager('@py')

# Connect using the instrument's IP address
scope = rm.open_resource('TCPIP::192.168.2.45::INSTR')

# Ask the instrument to identify itself (standard SCPI command)
print(scope.query('*IDN?'))

scope.close()

The *IDN? query is a standard SCPI command that every compliant instrument supports — it returns the manufacturer, model number, serial number, and firmware version. A good first test to confirm a connection is working.

What You Can Do With Each Instrument

Siglent SDS1202X-E — Oscilloscope

  • Capture screenshots of the display directly to the Mac — useful for experiment documentation
  • Pull waveform data off the scope as a CSV file for analysis in Python or a spreadsheet
  • Set timebase, voltage scale, trigger level, and coupling remotely via SCPI
  • Automate measurements (frequency, rise time, voltage, etc.) and log results to a file
  • Run the scope headlessly while logging data from a long-running experiment

Siglent SDG-2042X — Signal Generator

  • Set frequency, amplitude, waveform type (sine, square, pulse, ramp, noise, arbitrary) and DC offset from the Mac
  • Precisely configure a 0.2 Hz square wave for RC time constant experiments without touching the front panel
  • Sweep frequencies automatically — write a script that steps through 1 Hz, 5 Hz, 15.9 Hz, 50 Hz, 500 Hz, and 5 kHz and logs the oscilloscope reading at each step
  • Save and recall waveform configurations
  • Control both channels independently or synchronised

Siglent SDM3055 — Digital Multimeter

  • Send a READ? SCPI command and get a live measurement returned to the Mac instantly
  • Log voltage, current, or resistance readings over time to a CSV file
  • Watch an RC capacitor charge curve numerically while the scope shows it visually — both running simultaneously from one script
  • Automate repetitive measurements and export data for graphing

Rigol DP832 — DC Power Supply

Note: The DP832's LAN interface requires a paid licence that is not currently installed. It is controlled via USB instead. The SCPI commands are identical — only the connection method differs.

  • Set output voltage and current limits precisely on any of the three channels
  • Enable and disable output channels under software control
  • Log actual output voltage and current over time to verify supply accuracy
  • Automate power sequencing — turn on supply, wait, take a DMM reading, log the result

Combining All Four: Automated Test Scripts

Once all instruments are connected, a single Python script on the Mac can control all four together. For an RC time constant experiment, for example, a script could:

  • Tell the SDG-2042X to output a 0.2 Hz square wave at 5 V
  • Tell the DP832 to set the supply to exactly 5.000 V
  • Tell the SDM3055 to take a voltage reading every 100 ms and log it to a CSV file
  • Tell the SDS1202X-E scope to capture a screenshot at the end of the run

All automated, all from the Mac, with no front panel interaction needed. Here is a minimal example connecting to all three ethernet instruments and identifying them: 

import pyvisa

rm = pyvisa.ResourceManager('@py')

instruments = {
    'Oscilloscope':     'TCPIP::192.168.2.45::INSTR',
    'Signal Generator': 'TCPIP::192.168.2.46::INSTR',
    'Multimeter':       'TCPIP::192.168.2.20::INSTR',
}

for name, address in instruments.items():
    inst = rm.open_resource(address)
    idn = inst.query('*IDN?').strip()
    print(f"{name}: {idn}")
    inst.close()

Getting Everything Connected

  1. Plug the three Siglent instruments into the bench network switch via ethernet
  2. On each instrument, navigate to Utility → I/O → LAN and enable DHCP
  3. Run lxi discover on the Mac to confirm they appear with their IP addresses
  4. Connect the Rigol DP832 to the Mac via USB-B cable
  5. Run python -m scripts.utils.connection from the repo to confirm all four respond
  6. Type each ethernet instrument's IP address into Safari to confirm the built-in web interface loads
