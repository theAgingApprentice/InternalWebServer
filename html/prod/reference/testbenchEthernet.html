Test Bench — Ethernet Remote Control from Mac Studio

All four bench instruments have ethernet ports and support remote control from the Mac Studio over the local network. This page documents the protocol, software, and what can be done with each instrument remotely.

Instruments

  • Siglent SDS1202X-E — Oscilloscope
  • Siglent SDG-2042X — Signal Generator
  • Siglent SDM3055 — Digital Multimeter
  • Rigol DP832 — Programmable DC Power Supply

The Protocol: SCPI over LXI

All four instruments speak SCPI (Standard Commands for Programmable Instruments) — a universal command language for test equipment. Over ethernet they use the LXI standard (LAN eXtensions for Instrumentation), which means they all communicate the same way and can be controlled from the same software. Connect them all to the network switch and the Mac Studio can talk to all of them simultaneously.

Each instrument also has a built-in web interface — type its IP address into any browser on the Mac and a basic control panel appears immediately, with no software installation required.

Software: LXI Tools

The recommended free tool is LXI Tools — open source, runs natively on Mac, and explicitly supports all four instruments. Install it via Homebrew: 

brew install lxi-tools

LXI Tools includes both a command-line tool and a GUI. The GUI lets you discover all instruments on the network, send SCPI commands interactively, capture screenshots from instrument displays, and record data — all from one interface.

What You Can Do With Each Instrument

Siglent SDS1202X-E — Oscilloscope

  • Capture screenshots of the display directly to the Mac — useful for adding to experiment documents
  • Pull waveform data off the scope as a file for analysis in Python or a spreadsheet
  • Set timebase, voltage scale, trigger level, and coupling remotely via SCPI
  • Automate measurements (frequency, rise time, voltage, etc.) and log results to a file
  • Run the scope headlessly while logging data from a long-running experiment

Siglent SDG-2042X — Signal Generator

  • Set frequency, amplitude, waveform type (sine, square, pulse, etc.) and DC offset from the Mac
  • Precisely configure the 0.2Hz square wave for RC experiments without touching the front panel
  • Sweep frequencies automatically — write a script that steps through 1Hz, 5Hz, 15.9Hz, 50Hz, 500Hz, and 5kHz and logs the oscilloscope reading at each step
  • Save and recall waveform configurations

Siglent SDM3055 — Digital Multimeter

  • Send a READ? SCPI command and get a live measurement returned to the Mac
  • Log voltage, current, or resistance readings over time to a CSV file
  • Watch the RC capacitor charge curve numerically while the scope shows it visually — both running simultaneously from one script
  • Automate repetitive measurements and export data for graphing

Rigol DP832 — Programmable DC Power Supply

  • Set output voltage and current limits precisely from the Mac
  • Enable and disable outputs under software control
  • Log actual output voltage and current over time to verify supply accuracy
  • Automate power sequencing for experiments — e.g. turn on supply, wait, take DMM reading, log it

Combining All Four: Automated Test Scripts

Once all instruments are on the network, a Python script on the Mac can control all four together. For the RC time constant experiment, for example, a script could:

  • Tell the SDG to output a 0.2Hz square wave at 5V
  • Tell the DP832 to set the supply to exactly 5.000V
  • Tell the SDM to take a voltage reading every 100ms and log it to a file
  • Tell the SDS scope to capture a screenshot at the end of the run

All automated, all from the Mac, with no touching any front panel. This is how professional electronics labs run automated test sequences. The Python pyvisa library is the standard tool for this: 

pip install pyvisa pyvisa-py

First Steps: Getting Everything Connected

  • Plug all four instruments into the network switch via ethernet
  • On each instrument go to Utility → I/O → LAN and enable DHCP so they receive IP addresses automatically
  • Find each instrument's assigned IP address from its own front panel menu
  • Type each IP address into a browser on the Mac — the built-in web interface should appear immediately
  • Install LXI Tools via Homebrew and run a network discovery scan to confirm all four instruments appear
